# 仓颉 (Cangjie) v3.0 · 知识库驱动的高仿生写作引擎

> 「理以察势，文以载道，情以动人，意以通灵，神以进化。」

## 核心理念
仓颉 v3.0 的核心是 **「长出大脑」** 而非仅仅是「套用模板」。
它通过构建一个 **持久化的 Wiki (Persistent Wiki)**，让 AI 拥有与目标团队完全同步的知识背景、演化逻辑和修辞武器库：
- **理 (Logic)**：心智模型及其历史位移（思想的位移）。
- **文 (Expression)**：量化指纹 + 结构化隐喻库。
- **情 (Emotion)**：情感触发器与情感弧线建模。
- **意 (Intent/Wiki)** ⭐**v3.0 新增**：实体知识库，记录该团队对万物的全量历史看法。
- **神 (Evolution)**：分层 Correction 机制。

## 执行流程 (The Cangjie Wiki Swarm v3.0)

### Phase 0: 目标锚定与仓库初始化
- 创建 Wiki 式目录结构（Entities, Style_Fingerprints, Golden_Corpus）。
- 检索本地库，加载已有的知识复利。

### Phase 1: 增量式全网溯源 (Incremental Wiki Building)
启动 9+1 路并行 Agent，不仅采集语料，更进行 **「知识记账 (Bookkeeping)」**：
1. **实体 Agent** ⭐**核心**：识别语料中的核心实体（如：AI、王兴、SaaS），建立独立档案。
2. **演化 Agent**：识别观点随时间的变化，记录「演化拐点」。
3. **隐喻 Agent**：搜集并分类该团队偏好的类比、隐喻、梗。
4. **(原有 7 路 Agent 保持不变)**：著作、对话、表达、他者、决策、微信白名单、排版美学。

### Phase 2: 知识图谱化提炼 (Graph Synthesis)
1. **构建 Entity-Logic Map**：将心智模型链接到具体的实体上。
2. **提炼 Metaphor Vault**：建立结构化的类比仓库。
3. **识别 Internal Tensions**：记录并分析该团队观点中的内在矛盾。
4. **量化 Expression Fingerprints**：硬编码句式与标点参数。

### Phase 3: 意图驱动的仿生写作 (Intent-Based Rendering)
1. **加载 Wiki 上下文**：根据选题，调取相关的实体档案和演化记录。
2. **意译式重写**：理解巴菲特式「老友聊天」或晚点式「冷峻白描」的底层意图。
3. **注入隐喻**：强制从 Metaphor Vault 中调取匹配的修辞武器。

### Phase 4: 红蓝对抗审计与进化
1. **蓝军生成**：仓颉根据 Wiki 和 Skill 产出长文。
2. **红军审计**：独立 Agent 用 Stylometric 匹配度和「观点一致性」进行挑刺。
3. **反馈入库**：修正逻辑存入 `Correction.md`，并同步更新相关的实体 Wiki。

## 仓颉禁令 (v3.0 附加)
- 严禁忽略观点的历史演化（思想是流动的）。
- 严禁即兴发挥与团队调性不符的新类比（必须优先从隐喻库选取）。
- 严禁在 Wiki 冲突未解决前输出断言。
